/01 · install

One command. Any environment.

Skills follow the open SKILL.md spec — portable across Claude Code, Cursor, Codex, Gemini CLI, Windsurf, and any MCP-aware agent runtime.

$ claude plugin install basekit
$ cursor mcp add basekit
# or paste into .cursor/mcp.json:
{ "mcpServers": { "basekit": { "command": "npx", "args": ["-y", "@basekit/mcp"] } } }
$ codex mcp add basekit npx -- -y @basekit/mcp
$ windsurf mcp add basekit
# paste into ~/.codeium/windsurf/mcp_config.json:
{ "mcpServers": { "basekit": { "command": "npx", "args": ["-y", "@basekit/mcp"] } } }
$ git clone https://github.com/basekit/basekit
$ cd basekit && ./install.sh
# or copy ./skills/* into your agent's skill directory
After install: ask your agent "deploy an ERC-20 on Base sepolia" and the right skill activates automatically.
Before you install — read this
  • Your wallet always signs. BaseKit never touches your private key. Every state-changing action is drafted by a skill and signed by your wallet client.
  • Skills are advisory, not audited. Review the generated transaction before signing. Treat every skill like a knowledgeable colleague — useful, but you're the last line of defense.
  • Cost caps are enforced only with agent-onchain-handshake. The estimated_cost_usd_max field is a hint your agent reads. To actually bound spend, pair skills with the handshake delegator (EIP-7702 policy).
  • Keep API keys out of version control. Add this to your repo's .gitignore:
    .cursor/mcp.json · ~/.codeium/windsurf/mcp_config.json · .env*
  • Only install signed skills. v0.2 skills are signed: false — official but not yet cryptographically signed. Community skills will require signatures starting v0.3. See /security.

/03 · mcp

Thirteen tools your agent can call directly.

Skills give your agent playbooks. The BaseKit MCP server gives it arms. Server is read-only by design — mutating actions are described by skills, executed by the user's wallet.

verify_contract

Full safety report — source, owner, mint, pause, blacklist, GoPlus.

get_wallet_profile

Activity, balance, age, counterparties for a Base address.

portfolio_snapshot

ETH, ERC-20 holdings, recent activity — read-only.

simulate_swap

Live quote against Uniswap v3 Quoter. Does not execute.

check_token_approvals

Scan Approval events. Surface unlimited + dangerous spenders.

get_transaction_trace

Receipt + decoded selector + tracer links.

resolve_basename

Live L2 resolver. Forward and reverse.

check_basename_available

Availability + cost projection by name length.

gas_now

Current Base fee + sequencer health + send-now suggestion.

decode_calldata

Selector + signatures (4byte cross-check).

estimate_token_deploy

Preflight gas + ETH cost.

estimate_safe_deploy

Preflight Safe deploy cost by owner count.

agent_policy_describe

Render a user's signed policy for confirmation UI.

/04 · a2a

Safe for agents acting on your behalf.

The bet: agents will spend money on tools. They will only spend it on tools they can verify are safe. Every part of BaseKit is built around that.

01

Declared risk

Every skill ships YAML frontmatter: risk_level, reversible, requires_signing, mutates_state, estimated_cost_usd_max, external_calls. An agent reads these before deciding whether to proceed.

02

Read-only by default

The MCP server has no mutating tools. Every state change is described by a skill and executed by the user's wallet client. Compromising the server gets you nothing.

03

Signed user policy

Writes on a user's behalf require a signed policy: per-tx cap, per-day cap, allowed contracts, allowed skills, human-confirm threshold. Anchored onchain via EIP-7702 delegator.

04

Bounded execution

Session key, not your main key. Agent operates within the signed bounds — primary funds untouchable even if the agent is compromised.

05

Audit trail

Every action emits a signed log entry: skill, tx hash, policy hash, USD value. Designed to be diffable, queryable, and admissible if anything goes wrong.

06

Instant revoke

One transaction to revoke an agent. One nonce bump to invalidate every outstanding offchain grant. No customer support required.

# the contract an agent makes when it acts on your behalf
POST /.well-known/agent.json     → capability + trust descriptor
READ policy_hash from delegator  → verify the user's signed bounds
CHECK skill.frontmatter.risk     → abort if above user's threshold
SIMULATE call, price in USD      → abort if above per-tx cap
EXECUTE via session key          → primary key untouched
EMIT signed audit log entry      → skill, tx hash, USD value, policy hash
