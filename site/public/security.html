Security

What we protect against, what we don't, and how to report issues. Read this before delegating real money to an agent running BaseKit.

The promise (and its limits)

BaseKit's core security claim is simple: your wallet always signs. The MCP server has no mutating tools by design — it cannot send transactions, approve tokens, or move funds. Every state-changing action is described by a markdown skill and executed by your wallet client (Frame, Rabby, Metamask, Coinbase Wallet, etc).

What this means in practice: a fully compromised BaseKit MCP server — backdoored binary, malicious package update, hijacked process — still cannot drain your wallet. The attack surface for theft is your wallet client and your signing decisions, not us.

What this doesn't protect against: bad transactions you approve. If a skill recommends a transaction and you sign it, the transaction executes. We give you the metadata to decide (risk_level, reversibility, cost cap, external calls) — the decision is still yours.

Disclosure policy

If you find a vulnerability, report it privately. We do not run a bounty program yet, but we acknowledge every reporter publicly with their consent and prioritize fixes.

  • Email: security@basekit.dev
  • PGP key: coming with v0.3 (fingerprint will be published here and at /.well-known/security.txt)
  • Response time: initial acknowledgment within 48 hours, fix or mitigation within 14 days for critical issues
  • In scope: the MCP server, official skills, the delegator contract once deployed, and this website
  • Out of scope: user-side wallet clients, third-party APIs (Basescan, Alchemy), and code patterns demonstrated in skills that users modify before deploying

What we ask of you

  • Read the transaction before signing. Every wallet client shows you what you're approving. Don't skip it.
  • Start on Base Sepolia. Every skill works on testnet. Validate the flow with fake money first.
  • Use the handshake. If you're delegating to an autonomous agent, do it through agent-onchain-handshake with a session key bounded by an explicit policy. Don't give an agent your primary key.
  • Pin your dependencies. Reference a specific @basekit/mcp version in your config. Don't blindly accept updates.

What we don't promise

BaseKit is MIT-licensed, provided as-is. The skills encode our best understanding of safe patterns on Base, but they are not a substitute for understanding what you're signing. No skill, no MCP server, no agent framework removes the responsibility of the person whose wallet executes the final transaction.
